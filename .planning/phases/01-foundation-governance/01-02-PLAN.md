       - Function `parse_rego_rules(rego_path: pathlib.Path) -> Dict[str, List[Dict]]`:
         - Read .rego file text
         - Parse `deny contains msg if {` blocks and `warn contains msg if {` blocks
         - For each block, extract:
           - conditions (lines between `{` and `}`)
           - message pattern (the `msg := "..."` or `msg := sprintf(...)` line)
         - Return `{"deny": [...], "warn": [...]}` where each item is a dict with `conditions` and `msg_template`
         - Use brace-counting to handle nested blocks correctly

         - Convert the gatekeeper's collect_records() output into the input JSON contract:
           ```python
           {
               "records": [
                   {
                       "path": str(path.relative_to(ROOT)),
                       "frontmatter": frontmatter_dict,
                       "sections": {section_name: True for section_name in sections_list}
                   }
                   for path, frontmatter_dict, sections_list in records
               ]
           }
           ```

       - Function `evaluate_skill(skill_dir: pathlib.Path, input_data: Dict) -> RuleResult`:
         - Read `guidelines.md` frontmatter to get `skill_id`
         - Read `rules.rego` from skill_dir
         - Evaluate each deny/warn rule against input_data using Python logic:
           - `count(input.records) == 0` -> `len(input_data["records"]) == 0`
           - `some record in input.records` -> iterate over records
           - `not record.frontmatter.X` -> `X not in record["frontmatter"]` or `not record["frontmatter"].get(X)`
           - `record.frontmatter.type == "adr"` -> filter by type
           - `not record.sections.X` -> `X not in record["sections"]`
           - `not record.sections.X; not record.sections["Y"]` (AND) -> both must be absent
           - `msg := sprintf("... '%s' ...", [record.path])` -> format with actual values
         - The evaluator does NOT need a full Rego parser. It needs to handle the specific patterns present in the 4 existing .rego files:
           Pattern A: `count(input.records) == 0` (global count check)
           Pattern B: `some record in input.records; not record.frontmatter.FIELD` (field presence)
           Pattern C: `some record in input.records; record.frontmatter.type == "TYPE"; not record.sections.SECTION` (type-filtered section check)
           Pattern D: `adr_records := [r | some r in input.records; ...]; count(adr_records) == 0` (filtered count)
           Pattern E: Two `not record.sections.X` in same block = AND (both must be absent for deny)
         - Return RuleResult with collected deny and warn messages

       Per D-05: This evaluator is the official gate authority reading from Skill rules.
       Per D-08: If a .rego file cannot be parsed, treat it as a deny (fail closed).
